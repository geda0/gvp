Tracks D/E are copy + polish that only make sense once the structure exists.
- **Test-migration risk (load-bearing).** `test/labs-restructure.test.mjs` asserts the OLD split across THREE concerns — `normalizeSection` (S1), index.html markup (S2), navigation routes (S3). Each must be re-spec'd RED by the test-writer FIRST (ADR-0011 §"This breaks the existing IA contract tests") before the implementer makes the merged world pass; I split it into three slices so each is one honest red→green. Do NOT let the implementer "fix" the test to pass the old code.
- **Keep `data.playground` in the JSON.** ADR-0011 §6 + the merge slices (S5/S16/S17) keep BOTH `data.playground` and `data.portfolio` arrays — only the RENDER path concatenates. `test/team-tactics-project.test.mjs` (reads `data.playground[0]` for the featured card) and the team-tactics 0.56 copy test must stay GREEN unchanged through the whole milestone; a slice that flattens the data array would break them and is OUT OF SCOPE here.
- **index.html env-guard (S2).** The big index.html edit touches nav + section containers + hero, but MUST NOT disturb the `gvp:contact-api-url` / `gvp:chat-api-url` meta tags (ADR-0011 §"Env-guard / structural invariants — NOT touched"; the 2026-06-04 staging-host-leak incident). `test/frontend-api-url-env-guard.test.mjs` must stay green — called out on S2 and again on S18 (hero edit) since both touch index.html.
